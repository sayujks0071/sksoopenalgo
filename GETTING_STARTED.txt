🚀 DEPLOYMENT IN 5 STEPS
================================================================================

STEP 1: Access Your AWS Lightsail Instance
────────────────────────────────────────────
From your local machine:

  $ ssh -i /path/to/your/lightsail-key.pem ubuntu@YOUR_LIGHTSAIL_IP

Example:
  $ ssh -i ~/.ssh/lightsail.pem ubuntu@123.45.67.89


STEP 2: Clone the Repository
────────────────────────────────────────────
On the Lightsail instance:

  $ git clone https://github.com/marketcalls/openalgo.git
  $ cd openalgo


STEP 3: Run the Deployment Script
────────────────────────────────────────────
  $ chmod +x deploy_lightsail.sh
  $ ./deploy_lightsail.sh algo.endoscopicspinehyderabad.in

  ⏱️  This takes 5-15 minutes. It will:
      • Install Docker and dependencies
      • Configure Nginx with SSL
      • Set up PostgreSQL database
      • Create systemd service
      • Generate certificates
      • Start all containers


STEP 4: Update Credentials
────────────────────────────────────────────
After deployment completes:

  $ sudo nano /opt/openalgo/.env

  ⚠️  UPDATE THESE VALUES:
      BROKER_API_KEY=YOUR_DHAN_API_KEY
      BROKER_API_SECRET=YOUR_DHAN_API_SECRET
      DB_PASSWORD=STRONG_PASSWORD_25_CHARS

  Save file: Ctrl+O, Enter, Ctrl+X


STEP 5: Restart Services and Verify
────────────────────────────────────────────
  $ sudo systemctl restart openalgo
  $ /opt/openalgo/monitor.sh

  ✅ You should see all services running
  ✅ Health checks should pass
  ✅ Access: https://algo.endoscopicspinehyderabad.in

🔐 SECURITY NOTES
================================================================================

1. CREDENTIALS
   • Keep .env file permissions at 600 (read/write for owner only)
   • Use strong passwords (25+ characters, mixed case, numbers, symbols)
   • Rotate secrets every 90 days
   • Never commit .env file to git

2. FIREWALL
   • Allow port 80 for Let's Encrypt certificate renewal
   • Allow port 443 for HTTPS web access
   • Restrict SSH (port 22) to your IP only
   • Configure in AWS Lightsail console

3. BACKUPS
   • Create backups before updates
   • Test restore procedures monthly
   • Store backups securely

4. MONITORING
   • Check health daily: /opt/openalgo/monitor.sh
   • Review logs weekly
   • Monitor SSL certificate expiration

📞 TROUBLESHOOTING QUICK FIX
================================================================================

"Site not accessible"
  1. Check containers: docker compose ps
  2. Check logs: /opt/openalgo/manage.sh logs
  3. Check firewall: AWS Lightsail Console → Networking

"Certificate error"
  1. Wait 5-10 minutes for Let's Encrypt to issue cert
  2. Check: echo | openssl s_client -connect algo....:443
  3. Restart nginx: docker compose restart nginx

"Database connection failed"
  1. Test: /opt/openalgo/manage.sh psql
  2. Restart: docker compose restart postgres
  3. Check .env: grep "^DB_" /opt/openalgo/.env

"API not responding"
  1. Check logs: /opt/openalgo/manage.sh logs openalgo
  2. Restart app: /opt/openalgo/manage.sh restart openalgo
  3. Check resources: docker stats
